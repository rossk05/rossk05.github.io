What Are APIs? ü§î

API = Application Programming Interface

Think of an API as a messenger that takes your request, tells a system what you want, and brings back the response.

Restaurant Analogy:
‚Ä¢ You (client) want food
‚Ä¢ Waiter (API) takes your order
‚Ä¢ Kitchen (server) prepares food
‚Ä¢ Waiter brings food back to you

You don't need to know how the kitchen works - just how to order!

Why APIs Matter üí°

APIs let different software talk to each other. Without them:

  • Weather apps couldn't get weather data
  • Social media couldn't share posts
  • Payment systems couldn't process transactions
  • Maps couldn't show your location

Real Examples You Use Daily:

Instagram: Uses APIs to post photos, get your feed, send messages

Uber: Uses Google Maps API for navigation, payment APIs for billing

Weather Apps: Use weather service APIs to get current conditions

The API Request Journey

Every API call follows the same pattern: Request ‚Üí Process ‚Üí Response

Step 1: Choose Your Method

GET: Get data (like reading a book)

POST: Send new data (like mailing a letter)

PUT: Update existing data (like editing a document)

DELETE: Remove data (like throwing away trash)

Step 2: Build Your URL

https://api.example.com/users/123 ‚Üë ‚Üë ‚Üë ‚Üë protocol domain path resource

Step 3: Add Headers (if needed)

const headers = { 'Authorization': 'Bearer your-token-here', 'Content-Type': 'application/json' };

Step 4: Make the Request

// Modern way with fetch const response = await fetch('https://api.example.com/users', { method: 'GET', headers: headers }); const data = await response.json(); console.log(data);

Planning Your API üìã

Example: Todo List API

What it does: Manage todo items

Resources: Users, Todos

Actions: Create, Read, Update, Delete todos

Step 1: Design Your Endpoints

GET /api/todos ‚Üí Get all todos POST /api/todos ‚Üí Create new todo GET /api/todos/123 ‚Üí Get specific todo PUT /api/todos/123 ‚Üí Update specific todo DELETE /api/todos/123 ‚Üí Delete specific todo

Step 2: Choose Your Tech Stack

Node.js + Express
JavaScript, fast setup
Python + Flask
Simple, great for beginners
Python + FastAPI
Modern, auto-docs

Step 3: Write Your First Endpoint

const express = require('express'); const app = express(); // Simple GET endpoint app.get('/api/hello', (req, res) => { res.json({ message: 'Hello World!', timestamp: new Date() }); }); // POST endpoint with data app.post('/api/todos', (req, res) => { const { title, completed } = req.body; // In real app, save to database const newTodo = { id: Date.now(), title: title, completed: completed || false }; res.status(201).json(newTodo); }); app.listen(3000, () => { console.log('API running on port 3000! üöÄ'); });

Step 4: Add Error Handling

app.get('/api/todos/:id', (req, res) => { const todoId = req.params.id; // Validate ID if (!todoId || isNaN(todoId)) { return res.status(400).json({ error: 'Invalid todo ID' }); } // Find todo (simulate database) const todo = findTodoById(todoId); if (!todo) { return res.status(404).json({ error: 'Todo not found' }); } res.json(todo); });

Step 5: Document Your API
